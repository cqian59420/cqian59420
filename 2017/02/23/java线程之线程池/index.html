java线程之线程池

Executor框架

Executor框架的好处，其实就是线程池的好处，可以复用线程，降低每次新建线程的消耗。
结构上将任务体和执行策略分离开来,有助于在部署阶段选择与可用资源最匹配的执行策略.

可以使用Executors工具创建线程池

  • newFixedThreadPool 创建一个固定长度的线程池，提交一个任务就创建一个线程,直到达到
    线程池的最大尺寸
  • newCachedThreadPool 可缓存的线程池，如果当前线程规模超过处理需求，回收空闲线程，
    当需求增加时，增加新的线程。缓存线程池不存在任何限制.
  • newSingleThreadPool 单线程的线程池.如果线程异常出错，就创建一个新的线程.newSingleThreadPool
    按照队列中的顺序穿行执行.
  • newScheduledThreadPool 创建一个固定长度的线程池，并用延迟或者定时的方法来执行其中的线程.
